# TG Poster

Telegram-бот для автоматической публикации файлов (фото, видео, документы) в каналы по расписанию с веб-интерфейсом управления.

## 📋 Возможности

- **Веб-интерфейс** — управление очередями, каналами и мониторинг публикаций
- **Планировщик** — автоматическая публикация по времени (с учётом интервалов и jitter)
- **Гибкость** — настройка времени начала/окончания, интервала между постами
- **Мониторинг** — отслеживание статуса отправок, логов и ETA завершения
- **Тестирование** — отправка тестовых сообщений в каналы
- **Docker** — готовность к развёртыванию через Docker Compose
- **Настройки через веб** — управление токеном бота, часовым поясом и путями без редактирования .env
- **Умная отправка изображений** — файлы >2MB отправляются как сжатое фото + оригинал документом
- **Проверка лимитов Telegram** — автоматическая проверка размеров файлов перед отправкой

## 🚀 Быстрый старт

### Через Docker Compose (рекомендуется)

1. **Склонируйте репозиторий:**
   ```bash
   git clone <repository-url>
   cd tg_poster
   ```

2. **Настройте переменные окружения:**
   
   Создайте файл `.env` на основе `.env.example`:
   ```bash
   cp .env.example .env
   ```
   
   Отредактируйте `.env`, указав ваш токен бота:
   ```
   BOT_TOKEN=your-bot-token-here
   TZ=Asia/Yekaterinburg
   MEDIA_PATH=/app/media
   ```

3. **Запустите контейнер:**
   ```bash
   docker-compose up -d
   ```

4. **Откройте веб-интерфейс:**
   ```
   http://localhost:7777
   ```

5. **Настройте токен через веб-интерфейс:**
   - Перейдите в раздел **⚙️ Настройки**
   - Введите токен бота и сохраните

### Локальный запуск (без Docker)

1. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

2. **Настройте переменные окружения:**
   ```bash
   export BOT_TOKEN="your-bot-token"
   export TZ="Asia/Yekaterinburg"
   export MEDIA_PATH="/path/to/your/media"
   ```

3. **Запустите приложение:**
   ```bash
   python -m app.main
   ```

4. **Откройте в браузере:**
   ```
   http://127.0.0.1:7777
   ```

## 📖 Использование

### 1. Добавление канала

1. Перейдите на главную страницу (`/`)
2. Введите `chat_id` канала (можно получить через [@userinfobot](https://t.me/userinfobot))
3. Укажите название канала
4. Нажмите «Добавить»

> ⚠️ Бот должен быть администратором канала с правом публикации сообщений.

### 2. Создание очереди

1. Перейдите на страницу очередей (`/queues`)
2. Выберите канал из списка
3. Настройте параметры:
   - **Название** — имя очереди
   - **Папка источника** — путь к файлам для публикации
   - **Время начала/окончания** — диапазон публикации (формат `HH:MM`)
   - **Интервал** — минимальное время между постами (сек)
   - **Jitter** — случайная задержка (сек) для естественности
4. Нажмите «Создать очередь»

### 3. Управление очередью

На странице управления очередью (`/queue/<id>/manage`):

- **Старт/Пауза** — запуск или остановка публикации
- **Принудительный старт** — немедленная активация
- **Тестовая отправка** — проверка связи с каналом
- **Логи** — история отправок (последние 50)
- **Удаление** — удаление очереди и её логов

### 4. Настройки приложения

Перейдите в раздел **⚙️ Настройки** (`/settings`):

- **Telegram Bot Token** — токен бота для работы с API
- **Часовой пояс** — используется для расписания и логов
- **Пути к файлам** — базовая директория для медиафайлов
- **Управление каналами** — редактирование и удаление каналов

## 🔧 Конфигурация

### Переменные окружения

| Переменная | Описание | Пример |
|------------|----------|--------|
| `BOT_TOKEN` | Токен Telegram-бота | `123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11` |
| `TZ` | Часовой пояс | `Asia/Yekaterinburg` |
| `MEDIA_PATH` | Путь к медиафайлам | `/app/media` |

> 💡 Все настройки можно изменить через веб-интерфейс в разделе **⚙️ Настройки**.

### Параметры очереди

- **interval_sec** — базовый интервал между публикациями (мин. 10 сек)
- **jitter_sec** — случайная добавка к интервалу для вариативности
- **queue_order** — порядок обработки очередей (автоинкремент)

## 🗄 База данных

Приложение использует SQLite (`poster.db`) со следующими таблицами:

- **channels** — список каналов (id, chat_id, name)
- **queues** — очереди публикаций (параметры, статус, настройки)
- **post_log** — лог отправок (статус, ошибки, временные метки)

## 🆕 Реализованные функции

### Недавние улучшения

- ✅ **Конвертация AVIF в JPG** — автоматическое преобразование AVIF файлов при сканировании с перемещением оригиналов в папку `errors/` (используется pillow-heif для надёжной конвертации)
- ✅ **Уведомления об ошибках** — отправка уведомлений в личные сообщения разрешённым пользователям при ошибках отправки файлов
- ✅ **Управление разрешёнными пользователями** — веб-интерфейс для добавления/удаления User ID пользователей, получающих уведомления. При добавлении отправляется тестовое сообщение для проверки существования пользователя
- ✅ **Очистка истории канала** — кнопка для очистки логов отправленных файлов для каждого канала
- ✅ **Сортировка в таблице логов** — сортировка по столбцам: время (от старого к новому), название файла, тип, статус (по алфавиту)
- ✅ **Столбец "Файл" в логах** — отображение имени файла в таблице истории отправок
- ✅ **Асинхронная обработка очередей** — убран блокирующий `time.sleep()`, каждая очередь работает независимо через проверку `next_send_time` в БД
- ✅ **Исправление интервалов** — добавлено поле `next_send_time` в таблицу `queues` для точного контроля времени следующей отправки
- ✅ **Проверка размера файлов** — автоматическая проверка лимитов Telegram (10MB фото, 50MB видео/документы)
- ✅ **Умная отправка изображений >2MB** — сначала сжатое фото, затем оригинал как документ (через паузу 15-20 сек)
- ✅ **Веб-страница настроек** — управление токеном, часовым поясом и путями без редактирования файлов
- ✅ **Редактирование каналов** — изменение названия и chat_id через веб-интерфейс
- ✅ **Удаление токена из репозитория** — использование `.env` для чувствительных данных
- ✅ **Graceful shutdown планировщика** — корректная остановка при завершении работы
- ✅ **Увеличенные таймауты для больших файлов** — 300 сек для файлов >10MB, 180 сек для файлов >5MB
- ✅ **WebP отправляется как документ** — чтобы избежать отправки как стикер
- ✅ **Исключение Thumbs.db** — системные файлы Windows не отправляются
- ✅ **Исключение папок errors/sended** — служебные папки не отображаются в списке выбора источников
- ✅ **Валидация даты окончания очереди** — нельзя установить дату окончания раньше даты начала
- ✅ **Отображение ошибок подключения** — при добавлении канала выводятся понятные ошибки (VPN, таймаут, неверный chat_id)

### Существующие функции

- **Jitter (случайная задержка)** — добавление случайной вариации к интервалу для естественности публикаций
- **Приоритет очередей** — настройка порядка обработки очередей, возможность изменения порядка «вверх/вниз»
- **Принудительный запуск** — немедленная активация очереди вне зависимости от расписания
- **ETA завершения** — расчёт предполагаемого времени окончания публикации очереди
- **Веб-интерфейс с автообновлением** — мониторинг статусов в реальном времени
- **Тестирование каналов** — быстрая проверка связи с каналом перед запуском очереди
- **Гибкое расписание** — настройка времени начала/окончания публикаций для каждого канала
- **Автоматическое определение типа файла** — фото, видео, GIF, документ

## 📌 Как работает очередь постов

**Текущая реализация:**

- Планировщик проверяет активные очереди каждые 10 секунд
- Каждая активная очередь имеет поле `next_send_time` в БД
- При проверке: если текущее время < `next_send_time`, очередь пропускается
- После отправки файла вычисляется новое `next_send_time = now + interval + jitter`
- Интервалы соблюдаются точно, независимо от других очередей

**Преимущества:**

- ✅ Очередь канала А с интервалом 60 сек работает независимо
- ✅ Очередь канала Б с интервалом 30 сек работает параллельно
- ✅ Пауза/ошибка одной очереди не затрагивает остальные
- ✅ Точное соблюдение интервалов без блокирующих задержек
