# TG Poster

Telegram-бот для автоматической публикации файлов (фото, видео, документы) в каналы по расписанию с веб-интерфейсом управления.

## 📋 Возможности

- **Веб-интерфейс** — управление очередями, каналами и мониторинг публикаций
- **Планировщик** — автоматическая публикация по времени (с учётом интервалов и jitter)
- **Гибкость** — настройка времени начала/окончания, интервала между постами
- **Мониторинг** — отслеживание статуса отправок, логов и ETA завершения
- **Тестирование** — отправка тестовых сообщений в каналы
- **Docker** — готовность к развёртыванию через Docker Compose

## 🚀 Быстрый старт

### Через Docker Compose (рекомендуется)

1. **Склонируйте репозиторий:**
   ```bash
   git clone <repository-url>
   cd tg_manager/tg_poster
   ```

2. **Настройте переменные окружения:**
   
   Отредактируйте `docker-compose.yml`:
   - Замените `BOT_TOKEN` на токен вашего бота
   - При необходимости измените пути к медиафайлам (`C:/tg_media`)
   - Измените порт маппинг если нужно (`7777:7777`)

3. **Запустите контейнер:**
   ```bash
   docker-compose up -d
   ```

4. **Откройте веб-интерфейс:**
   ```
   http://localhost:7777
   ```

### Локальный запуск (без Docker)

1. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

2. **Настройте переменные окружения:**
   ```bash
   export BOT_TOKEN="your-bot-token"
   export TZ="Asia/Yekaterinburg"
   ```

3. **Запустите приложение:**
   ```bash
   python -m app.main
   ```

4. **Откройте в браузере:**
   ```
   http://127.0.0.1:7777
   ```

## 📖 Использование

### 1. Добавление канала

1. Перейдите на главную страницу (`/`)
2. Введите `chat_id` канала (можно получить через [@userinfobot](https://t.me/userinfobot))
3. Укажите название канала
4. Нажмите «Добавить»

> ⚠️ Бот должен быть администратором канала с правом публикации сообщений.

### 2. Создание очереди

1. Перейдите на страницу очередей (`/queues`)
2. Выберите канал из списка
3. Настройте параметры:
   - **Название** — имя очереди
   - **Папка источника** — путь к файлам для публикации
   - **Время начала/окончания** — диапазон публикации (формат `HH:MM`)
   - **Интервал** — минимальное время между постами (сек)
   - **Jitter** — случайная задержка (сек) для естественности
4. Нажмите «Создать очередь»

### 3. Управление очередью

На странице управления очередью (`/queue/<id>/manage`):

- **Старт/Пауза** — запуск или остановка публикации
- **Принудительный старт** — немедленная активация
- **Тестовая отправка** — проверка связи с каналом
- **Логи** — история отправок (последние 50)
- **Удаление** — удаление очереди и её логов

## 🔧 Конфигурация

### Переменные окружения

| Переменная | Описание | Пример |
|------------|----------|--------|
| `BOT_TOKEN` | Токен Telegram-бота | `123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11` |
| `TZ` | Часовой пояс | `Asia/Yekaterinburg` |

### Параметры очереди

- **interval_sec** — базовый интервал между публикациями (мин. 10 сек)
- **jitter_sec** — случайная добавка к интервалу для вариативности
- **queue_order** — порядок обработки очередей (автоинкремент)

## 🗄 База данных

Приложение использует SQLite (`poster.db`) со следующими таблицами:

- **channels** — список каналов (id, chat_id, name)
- **queues** — очереди публикаций (параметры, статус, настройки)
- **post_log** — лог отправок (статус, ошибки, временные метки)

## 🆕 Новые функции

- **Поддержка альбомов** — автоматическое объединение файлов в альбомы при отправке. Приложение сканирует все вложенные подпапки источника ищет папки с названиями, содержащими «альбом», «album» или «новая папка», и отправляет их содержимое одним постом-альбомом.
- **Улучшенный подсчёт прогресса** — обновлённая логика расчёта количества постов с учётом альбомов. Общее количество поддерживаемых файлов в исходной папке минус количество папок-альбомов, умноженное на количество файлов в них, равно итоговому количеству постов.
- **Управление подписями через бота** — приём, сохранение и удаление подписей к постам напрямую в личных сообщениях с ботом Telegram.
- **Применение сохранённых подписей** — возможность добавлять ранее сохранённые подписи к новым постам.
- **Добавление постов в очередь через бота** — функция приёма постов для добавления в очередь публикации напрямую из личного чата с ботом.
